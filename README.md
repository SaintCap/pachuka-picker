# 🎮 ПАЧУКА // Терминал выбора игры

Сайт для выбора игры: пользователь логинится, читает зашифрованные описания,
выбирает одно, жмёт «Выбор сделан» — и с эффектами узнаёт название игры
и получает ссылку на её страницу в Steam. Выбор запоминается — при
следующем визите эта игра больше не предлагается.

Список игр и учётные записи пользователей хранятся в **Supabase**
(Postgres + Auth). Сам сайт остаётся статическим (HTML/CSS/JS без сборки),
Supabase-ключи подставляются во время билда на Render.

## Структура проекта

| Файл                  | Что это                                                                   |
|-----------------------|----------------------------------------------------------------------------|
| `index.html`          | Разметка экранов (загрузка, сбой, вход, главная, список, раскрытие, архив) |
| `style.css`           | Оформление и анимации                                                     |
| `src/`                | Логика приложения, разбитая на ES-модули (см. ниже)                       |
| `vendor/supabase.js`  | Локальная копия supabase-js — основной источник SDK                       |
| `supabase_setup.sql`  | SQL-скрипт для настройки таблиц, функций и прав доступа в Supabase         |
| `build-config.sh`     | Генерирует `config.js` из переменных окружения на Render перед публикацией |
| `config.example.js`   | Шаблон конфига для локального запуска                                     |
| `tests/`              | Сценарные тесты на jsdom (`npm test`) и бенчмарк отрисовки                 |

### Модули

Шага сборки нет — браузер грузит `src/main.js` как нативный ES-модуль.
Именно поэтому сайт нужно открывать **через http://**, а не двойным кликом
по `index.html`: с протокола `file://` модули не работают.

```
src/
  main.js              точка входа, только склейка модулей
  config.js            чтение и нормализация конфигурации
  sdk.js               загрузка supabase-js с откатом на резервные источники
  router.js            экраны поверх History API + охранные правила
  dom.js, env.js       мелкие утилиты и свойства среды
  data/
    client.js          singleton клиента, распознавание ошибок
    auth.js            вход, регистрация, профиль
    games.js           каталог, сохранение выбора, архив
  ui/
    cards.js           карточки досье
    authScreen.js      форма входа
    revealScreen.js    раскрытие названия
    historyScreen.js   архив выборов
    soundToggle.js     переключатель звука
    toast.js           всплывающие уведомления
  fx/
    engine.js          единый RAF-цикл, канвасы, DPR, ресайз
    sprites.js         предзапечённое свечение
    stars.js           звёздное небо
    particles.js       шлейф курсора и салют
    cursor.js          кастомный курсор
    audio.js           синтезатор звуков и режим тишины
    scramble.js        эффект расшифровки текста
```

Правило разделения: `data/` ничего не знает про DOM, `ui/` ничего не знает
про Supabase, `fx/` не знает ни о том, ни о другом. Связывает их только
`main.js` — поэтому, например, замена бэкенда затрагивает лишь `data/`.

### Почему SDK лежит локально

`vendor/supabase.js` — закоммиченная копия UMD-бандла `@supabase/supabase-js`.
Раньше SDK грузился только с jsdelivr, и если CDN был недоступен, скрипт падал
на первой же строке: пропадал не только вход, но и весь интерфейс — страница
превращалась в мёртвую картинку без курсора. Теперь `src/sdk.js` пробует источники
по очереди (локальный → jsdelivr → unpkg), а если не поднялся ни один —
показывает экран сбоя с кнопкой «Повторить».

Обновить копию:

```bash
npm pack @supabase/supabase-js@2
tar -xzO package/dist/umd/supabase.js < supabase-supabase-js-*.tgz > vendor/supabase.js
```

Версии в списке резервных CDN внутри `src/sdk.js` (`SDK_SOURCES`) стоит держать
в согласии с версией локальной копии.

## 1. Настройка Supabase

1. Зайди в свой проект на [supabase.com](https://supabase.com).
2. Открой **SQL Editor → New query**, вставь туда содержимое файла
   `supabase_setup.sql` целиком и нажми **Run**. Это создаст три таблицы:
   - `profiles` — логин (он же никнейм) пользователя;
   - `games` — список игр (наполняешь вручную);
   - `selections` — какие игры кто уже выбрал.

   Плюс функцию `get_available_games` для серверного отбора игр и
   Row Level Security, чтобы каждый видел только свои данные.
   Скрипт безопасно выполнять повторно — если база уже настроена,
   просто прогони его ещё раз, чтобы добавилась функция.
3. Открой **Authentication → Providers → Email** и **выключи "Confirm email"**.
   Это обязательно: сайт использует логин (не e-mail) и генерирует
   технический адрес вида `логин@pachuka.local`, подтвердить который
   письмом невозможно. Без этого шага регистрация не будет работать.
4. Возьми ключи проекта: **Project Settings → API**:
   - `Project URL` → это значение для `SUPABASE_URL`;
   - `anon public` ключ → это значение для `SUPABASE_ANON_KEY`.
5. Добавь игры: **Table Editor → games → Insert row**. Поля:
   - `name` — настоящее название игры;
   - `description` — шуточное/зашифрованное описание без названия;
   - `steam_url` — ссылка на страницу игры в Steam.
   Игр может быть сколько угодно, на экран выборки за раз выводится
   не больше 6 (и только те, что пользователь ещё не выбирал).

## 2. Настройка Render

Переменные окружения `SUPABASE_URL` и `SUPABASE_ANON_KEY` уже созданы —
впиши в них значения из шага 1.4 (Project Settings → API).

Дальше нужно, чтобы Render перед публикацией сайта подставил эти значения
в `config.js` (сам файл в репозитории не хранится, генерируется при билде):

1. Открой сервис на Render → **Settings**.
2. **Build Command:** `bash build-config.sh`
3. **Publish Directory:** `.` (без изменений)
4. Сохрани и сделай **Manual Deploy → Deploy latest commit** (или просто
   запушь любой коммит — задеплоится автоматически).

Если `SUPABASE_URL`/`SUPABASE_ANON_KEY` не заданы, билд специально
завершится с ошибкой — это защита от случайного деплоя без ключей.

## Как добавить/изменить игры

Через **Supabase → Table Editor → games**: добавляй, редактируй или
удаляй строки напрямую там. Никаких коммитов и деплоев для этого не нужно —
изменения появятся у пользователей сразу же.

## Как это работает

- Пользователь регистрируется (логин-никнейм и пароль) или логинится по
  логину и паролю. Технически это обычная Supabase Auth, где e-mail
  подставляется автоматически из логина.
- Логин он же никнейм: он показывается вместо «ПАЧУКА» на главном экране.
  Поэтому при регистрации поле подписано «Логин (никнейм)».
- При заходе в выбор отбор игр делает Postgres: функция
  `get_available_games` отсеивает уже выбранное, тасует остаток и отдаёт
  до 6 строк. Раньше клиент выкачивал весь каталог и фильтровал сам,
  а список исключений уезжал строкой прямо в URL запроса.
  Если функция на базе ещё не создана, клиент молча откатывается на
  старый путь — так что обновлять код и SQL можно не одновременно.
- После нажатия «Выбор сделан» запись `{ user_id, game_id }` сохраняется
  в `selections` — при следующем визите эта игра больше не появится.
- Экран «мои выборы» показывает архив: что выбрано и когда.
- Если игр нет вовсе или пользователь выбрал уже все — экран показывает
  сообщение «Игры закончились». Сбой сети показывается отдельным сообщением
  с кнопкой повтора, а не выдаётся за конец списка.
- Экраны отражаются в адресной строке, поэтому «Назад» и свайп на телефоне
  ходят внутри сайта, а не выкидывают из него. Экраны, на которые нельзя
  попасть по прямой ссылке (раскрытие без выбранной игры, что угодно без
  входа), охранное правило перенаправляет само.
- Если сессию отозвал сервер (протух токен, выход в другой вкладке), сайт
  сам возвращает на экран входа с пояснением.

## Локальный запуск

1. Скопируй `config.example.js` в `config.js` и впиши свои
   `SUPABASE_URL` / `SUPABASE_ANON_KEY` (файл в `.gitignore`, в репозиторий
   не попадёт).
2. Запусти локальный сервер (обязательно — из-за ES-модулей):
   ```bash
   python3 -m http.server 8000
   # открой http://localhost:8000
   ```

## Тесты

Сценарные тесты гоняют настоящие `index.html` и модули из `src/` в jsdom с подменённым
Supabase и проверяют поведение при сбоях: недоступный CDN, отсутствующий


Тесты не требуются для деплоя — Render публикует статику как есть.

### Бенчмарк отрисовки

`tests/bench.html` — открывается прямо в браузере, без сервера. Сравнивает
старый способ рисования свечения (`shadowBlur` на каждую фигуру каждый кадр)
с текущим (предзапечённый спрайт + `drawImage`) при одинаковой нагрузке
и показывает разницу в миллисекундах на кадр.

## Как устроена графика

Весь визуальный слой крутится в **одном** `requestAnimationFrame`-цикле:
звёзды, частицы и догоняющее кольцо курсора. Несколько решений, важных
для производительности:

- **Свечение — спрайты, а не `shadowBlur`.** Ореол один раз запекается
  в маленький офскрин-канвас, в кадре остаётся только `drawImage`.
  `shadowBlur` — самая дорогая операция Canvas 2D, и она выполнялась
  для каждой из сотен фигур каждый кадр.
- **Аддитивное смешивание** (`globalCompositeOperation = "lighter"`) —
  неон складывается как настоящий свет и стоит дешевле размытия.
- **Пустой FX-слой не перерисовывается.** Пока частиц нет (экраны входа
  и главной), канвас не трогается вообще.
- **Цикл останавливается на скрытой вкладке** через `visibilitychange`.
- **Ресайз с дебаунсом 150 мс**, звёзды масштабируются, а не создаются
  заново: на мобильных `resize` срабатывает при каждом появлении
  адресной строки.
- **Учёт `devicePixelRatio`** с потолком 2× — чётко на Retina без
  квадратичного роста цены.
- **Потолок в 1400 частиц**, отсев мёртвых компактизацией массива
  на месте, без создания нового массива каждый кадр.

## Фишки

- Кастомный неоновый курсор со шлейфом частиц
- Глитч-заголовок, летящие звёзды, ретро-сетка в стиле synthwave
- 3D-наклон карточек за мышкой
- Взрыв частиц, вспышка и тряска экрана при раскрытии
- Эффект «расшифровки» названия игры (scramble-текст)
- Синтезированные звуковые эффекты (Web Audio, без файлов)
- Адаптив под мобильные + уважение к `prefers-reduced-motion`
- Авторизация по логину/паролю и персональный список игр на базе Supabase
- Звук выключается одной кнопкой, выбор запоминается
- Навигация с клавиатуры с видимым фокусом, карточки читаются скринридером
- Архив выбранных игр с датами и ссылками
